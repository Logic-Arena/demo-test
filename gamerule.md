[게임 흐름]

웹사이트 접속 -> 게임페이지 및 채팅방목록이 보임 방만들기도 가능 -> 방을 만들거나 채팅방 참여 -> 다른사람이 들어옴 (사람은 1대1)-> ai가 주제를 던져줌 각자 찬성, 반대를 고를수 있음 10초동안의 시간을 줌 -> 찬성이나 반대가 겹칠경우 다시 ai가 주제를 던져주고 3번 겹치게 될 경우 주제와 찬성, 반대가 랜덤하게 결정된다.

-> 게임은 찬성측 사람1명 ai 한명 , 반대측 사람 한명 ai 한명 총 4인으로 구성되며 시작된다

-> 시작은 주장시간 2분을 각각 갖게 되며 자신의 주장을 작성한다 2분이 지나면 각각의 주장이 카드형태로

찬성측 사람 주장카드 , ai 주장카드 , 반대측 사람 주장카드, ai 주장카드를 화면에 나오며 모두가 읽을수 있는 상태가 된다 -> 찬성측 사람부터 바로 반대측 주장카드를 읽고 반론을 제기하며 제한시간은 2분이다,반론도 카드형태로 모두가 읽을 수 있게 나온다(반론을 처음 하는 사람은 주장 읽을시간이 부족하여 2분이고 반대측 반론을 할땐 1분30초로 해야함) -> 반론이 끝나면 반대측 사람과 ai가 변론을 한다 이 역시 카드로 제출하여 보이게 되는 방식이며 시간은 1분30초 이다.-> 변론을 읽고 재반론 시간을 1분 갖는다 (이것이 1사이클)-> 반대측 사람도 똑같이 찬성측 주장을 반론(1분30초)하면 찬성측 사람과,ai가 변론하고(1분30초), 다시 반대측 사람이 재반론을 한다(1분) 이렇게 2사이클 -> 이제 찬성측 ai도 10초간 반대측 주장에대해 반론하고 똑같이 반대측 사람과 ai가 변론하고 찬성측 ai가 재반론을한다. 이렇게 3사이클 마지막으로 반대측 ai가 찬성 주장에 대해 반론하고 찬성측 사람과 ai가 변론하고 반대측ai가 재반론을 하며 4사이클이 돌고 마지막으로 각자 1분씩 최종자기의견을 적고 제출하면서 게임이 끝난다 게임이 끝나면 ai심판이 논리점수를 수치화 하여 제공해주고 사람들에게 피드백을 해준다

## Project Overview
사용자 2명과 AI 2명이 팀을 이루어(2:2) 특정 주제에 대해 토론하는 웹 게임을 개발합니다. 실시간 채팅, 타이머 로직, 카드 형태의 UI, 그리고 AI 심판의 피드백 기능이 포함됩니다.

## Tech Stack
- Frontend: Next.js (App Router), Tailwind CSS, Lucide React (Icons)
- State Management: React Context API or Zustand
- Real-time/Backend: Supabase (Realtime, Auth, Database)
- AI Integration: OpenAI API (GPT-4o) or Anthropic API (Claude 3.5 Sonnet)

## Core Game Flow & Logic

### 1. Lobby & Room Management
- 메인 페이지에서 방 목록 확인 및 방 만들기 기능.
- 1:1 매칭 시스템 (방에 2명이 입장하면 게임 시작 프로세스 진입).

### 2. Topic & Role Selection
- AI가 주제를 제안하고, 10초 내에 찬성(Pro)/반대(Con) 선택.
- 선택이 겹치면 주제 재제안 (최대 3회).
- 3회 연속 겹칠 경우 AI가 주제와 역할을 강제로 랜덤 배정.
- 최종 팀 구성: [찬성: 사람A + AI_A] vs [반대: 사람B + AI_B]

### 3. Debate Structure (The 4 Cycles)
모든 발언은 제한시간 종료 후 '카드' 형태로 화면에 고정되어 모두가 읽을 수 있어야 함.

- **Phase 0 (주장):** 전원 각자 2분간 주장 작성 및 제출.
- **Cycle 1 (사람A 반론):** 사람A가 반대측 카드 읽고 반론 (2분) -> 반대측(사람B, AI_B) 변론 (1분 30초) -> 사람A 재반론 (1분).
- **Cycle 2 (사람B 반론):** 사람B가 찬성측 카드 읽고 반론 (1분 30초) -> 찬성측(사람A, AI_A) 변론 (1분 30초) -> 사람B 재반론 (1분).
- **Cycle 3 (AI_A 반론):** AI_A 반론 (10초 자동 생성) -> 반대측 변론 (1분 30초) -> AI_A 재반론 (1분).
- **Cycle 4 (AI_B 반론):** AI_B 반론 (10초 자동 생성) -> 찬성측 변론 (1분 30초) -> AI_B 재반론 (1분).
- **Phase 5 (최종 의견):** 전원 1분간 최종 정리 발언 제출.

### 4. Judging & Feedback
- 게임 종료 후 AI 심판이 전체 토론 로그를 분석.
- 논리 점수 수치화 및 각 참가자에 대한 구체적인 피드백 제공.

## Design Requirements
- 카드 기반의 깔끔한 UI/UX.
- 현재 누구의 차례인지 보여주는 상태 표시기 및 실시간 카운트다운 타이머.
- 토론 카드는 스택 형태로 쌓이거나 리스트 형태로 가독성 있게 배치.
